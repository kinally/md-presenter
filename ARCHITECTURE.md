# md-presenter — 架构设计文档

> 文稿脚本 + Markdown 视觉内容 → 可视化的段落对齐 → 自动朗读的 PPT 风格 HTML 演示页

---

## 一、核心理念

**两条独立轨道，通过可视化界面手动对齐：**

```
┌────────────────────────────────────────────────────────┐
│                       用户                             │
└──┬─────────────────────────────────────────────────┬──┘
   │                                                 │
   ▼                                                 ▼
┌───────────────┐                           ┌───────────────┐
│  ① 导入脚本     │                           │  ② 导入 Markdown│
│  (纯文本，TTS读) │                           │  (视觉 PPT 内容) │
└───────┬───────┘                           └───────┬───────┘
        │                                           │
        ▼                                           ▼
┌───────────────┐                           ┌───────────────┐
│  ③ 点选分节     │                           │  ④ 解析标题结构 │
│  系统插入标签    │                           │  按 ## 分节     │
│  显示"第n节"预览 │                           │               │
└───────┬───────┘                           └───────┬───────┘
        │                                           │
        └──────────┬────────────────────┬───────────┘
                   │                    │
                   ▼                    ▼
        ┌──────────────────────────────────────┐
        │        ⑤ 对齐编辑器                    │
        │                                        │
        │  ┌────────────┐    ┌────────────┐     │
        │  │ 脚本段落    │ ←→ │ Markdown节  │     │
        │  │ 第1节       │    │ 引言        │     │
        │  │ 第2节       │    │ 市场背景     │     │
        │  │ (空白)      │    │ 方法论       │     │  ← 允许空白
        │  │ 第3节       │    │ (空白)      │     │
        │  └────────────┘    └────────────┘     │
        │  ▲ 拖拽匹配 / 插入空白 / 拆分页面      │
        └──────────────────┬───────────────────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │        ⑥ 自动滚播                      │
        │  每对齐一对 → 显示幻灯片 + TTS 朗读    │
        │  读完自动跳到下一对 → 直到结束          │
        └──────────────────────────────────────┘
```

**关键原则：**
- **脚本轨道** = TTS 要读的完整叙述文本，按语义分节
- **幻灯片轨道** = 画面上要显示的精简/图形化内容，按 Markdown 标题分节
- **两者通过可视化界面**对齐，而非通过约定语法自动匹配
- **两边都可以插入空白**：
  - 有标题但无朗读文本 → 脚本侧留空，幻灯片侧正常显示，停留几秒
  - 一段脚本太长了 → 对应多个幻灯片页面，翻页时 TTS 继续朗读不中断

## 三、核心数据模型

### ScriptSection（脚本段落）

由用户点选分节产生：

```typescript
interface ScriptSection {
  id: string
  /** 自动编号，如 "第1节" "第2节" …… */
  label: string
  /** 该节的文本内容 */
  text: string
  /** 预览片段（前 50 字） */
  preview: string
}
```

**来源：** 用户粘贴纯文本 → 系统按段落间距给出默认分节 → 用户点击调整分节点。

### SlideSection（Markdown 节）

由 MD 解析器按标题产生：

```typescript
interface SlideSection {
  id: string
  /** 来自 ## 标题文本，如 "市场规模" */
  heading: string
  /** 该标题下的所有渲染节点（一页或多页） */
  slides: SlideNode[]
}

interface SlideNode {
  id: string
  type: 'title-slide' | 'section-header' | 'paragraph' | 'code' | 'list' | 'table' | 'image'
  content: string          // 渲染用的 Markdown / HTML
  /** 是否需要朗读（代码块/表格不朗读） */
  readable: boolean
  /** 停留时长（秒），仅对不可读的节点生效 */
  holdSeconds: number
}
```

**说明：** 一个 `SlideSection` 可以包含多个 `SlideNode`。一段脚本太长时，一个 `ScriptSection` 会对应同一个 `SlideSection` 的多张 slide。

### AlignedPair（对齐后的播放单元）

这是最终的播放驱动数据：

```typescript
interface AlignedPair {
  id: string
  /** 对应的脚本段落 ID，null 表示无朗读 */
  scriptId: string | null
  /** 要朗读的文本，从 ScriptSection.text 来 */
  scriptText: string
  /** 对应的幻灯片节 ID */
  slideSectionId: string
  /** 该节包含的幻灯片页列表 */
  slides: SlideNode[]
  /** 是否由同一个 ScriptSection 拆分而来（同一段脚本跨多页） */
  isContinuedScript: boolean
}
```

渲染时 `AlignedPair[]` 就是播放列表。用户在对齐编辑器中操作的最终产物就是它。

### 对齐关系图

```
脚本轨道:
  第1节 ─────────────┐
  第2节 ─────────┐   │
  第3节 ───┐     │   │
  (空白)   │     │   │
            │     │   │
            ▼     ▼   ▼
幻灯片轨道:
  [引言]   [市场背景]  [方法论]  [结论]
     1        2        3       4

      ↓ 对齐后 ↓

AlignedPair[]:
  ┌────┬──────────┬──────────┬──────────┬──────────┐
  │ #  │ scriptId │ slideSec │ 朗读?    │ 说明      │
  ├────┼──────────┼──────────┼──────────┼──────────┤
  │ 1  │ 第1节    │ 引言     │ ✅       │          │
  │ 2  │ 第2节    │ 市场背景  │ ✅       │          │
  │ 3  │ null     │ 方法论   │ ❌       │ 无朗读    │
  │ 4  │ 第3节    │ 结论     │ ✅       │          │
  └────┴──────────┴──────────┴──────────┴──────────┘
```

## 四、模块划分

### 模块 1：脚本分节编辑器

| 项目 | 说明 |
|------|------|
| 职责 | 用户导入纯文本，通过点选/拖拽进行语义分节 |
| 输入 | 纯文本字符串 |
| 输出 | `ScriptSection[]` |
| 交互 | 每段文本前显示「+ 分节」按钮，点击即在此处切开 |

**流程：**
1. 用户粘贴或拖入 .txt 文件
2. 系统按双换行符 `\n\n` 给出默认分节候选
3. 每两个段落之间有一个细分割线 + 「合并 ↑」「分割 ↓」按钮
4. 左侧实时显示节编号和预览：「第1节：在当今快速发展的市场环境下……」
5. 用户可以拖拽调整顺序、删除空节、合并相邻节

### 模块 2：MD 解析器

| 项目 | 说明 |
|------|------|
| 职责 | 将 Markdown 按标题解析为 `SlideSection[]` |
| 输入 | Markdown 字符串 |
| 输出 | `SlideSection[]` |
| 依赖 | `marked` 或 `remark` |

**分段规则：**
- `#` / `##` / `###` 每遇到一个就切分新节
- 该标题后的所有内容归入该节，直到遇到下一个同级或上级标题
- 每个节内的内容再细拆为 `SlideNode[]`（段落、列表、代码块、表格等）

### 模块 3：对齐编辑器 ★ 核心

| 项目 | 说明 |
|------|------|
| 职责 | 让用户将脚本段落与 Markdown 节可视化对齐 |
| 输入 | `ScriptSection[]` + `SlideSection[]` |
| 输出 | `AlignedPair[]` |
| 布局 | 双列对比视图 |

**UI 布局：**

```
┌─────────────────────────────────────────────────────────┐
│  ← 脚本段落                          Markdown 幻灯片 →  │
├──────────────────────┬──────────────────────────────────┤
│  第1节               │  ## 引言                         │
│  "在当今快速发展的…"  │  ● 市场增长 30%                  │
│                      │  ● 客户需求转向个性化            │
│  ── ── ── ── ──     │  ── ── ── ── ── ──              │
│  [对齐到 ▼]          │  [对齐到 ▼]                      │
├──────────────────────┼──────────────────────────────────┤
│  第2节               │  ## 市场背景                     │
│  "根据最新报告…"     │  2024年全球市场规模…              │
├──────────────────────┼──────────────────────────────────┤
│  [+ 插入空白]        │  ## 方法论                       │
│                      │  (无脚本对应，仅显示)             │
├──────────────────────┼──────────────────────────────────┤
│  第3节               │  [+ 插入空白]                    │
│  "综上所述…"         │  (有脚本但无幻灯片？不常见但支持)  │
└──────────────────────┴──────────────────────────────────┘
```

**交互能力：**
- **拖拽连线**：从左侧脚本段落拖到右侧 Markdown 节，建立映射
- **插入空白**：两侧都有「+ 插入空白」按钮，插入一个空节占位
- **拆分页面**：在 Markdown 侧点击「+ 拆页」，将一节拆成多页（脚本不变，TTS 不中断）
- **自动匹配建议**：系统尝试按文本相似度给出初始匹配建议
- **验证提示**：未匹配的节用红色高亮，提示用户处理
- **序号联动**：对齐后统一编号，显示为播放时的顺序

### 模块 4：幻灯片渲染器

| 项目 | 说明 |
|------|------|
| 职责 | 把 `SlideNode[]` 渲染为全屏 HTML 幻灯片 |
| 技术 | Vue 3 + CSS `@keyframes` |
| 主题 | 内置 2~3 套：浅色、深色、简约 |

**每张 slide 的结构：**

```
┌─────────────────────────────┐
│  [章节标题]  (可选)           │
│                             │
│        主体内容区             │
│    (文字/列表/代码/表格)      │
│                             │
│  [页码/进度条]               │
└─────────────────────────────┘
```

**入场动画：**
- 标题 slide：文字淡入 + 上浮
- 正文 slide：从左向右划入，或 fade-in
- 代码/表格 slide：静止显示

**代码高亮：**
- `highlight.js` 按需加载

### 模块 5：TTS 接口层（策略模式）

```typescript
interface TTSProvider {
  readonly name: string
  readonly type: 'system' | 'api' | 'local'
  speak(text: string, onEnd: () => void): void
  pause(): void
  resume(): void
  cancel(): void
  get isSupported(): boolean
}
```

用户可以在设置面板中选择自己偏好的朗读方式。系统提供一个统一的切换下拉菜单。

**内置实现：**

| 实现类 | type | 机制 | 适用场景 |
|--------|------|------|----------|
| `WebSpeechTTS` | `system` | `window.speechSynthesis` | 所有现代浏览器，零配置 |
| `AiApiTTS` | `api` | 调 AI API 的 `/v1/audio/speech` 端点 | 用户有 OpenAI 兼容 API 且支持 TTS |
| `LocalModelTTS` | `local` | HTTP 请求本地运行的 TTS 服务端（如 GPT-SoVITS、RVC、xTTS） | 用户有 N 卡，跑本地推理 |
| `WinSapiTTS` | `system` | WebSocket / NamedPipe 调 Windows SAPI | Windows 特定场景（预留） |

#### AiApiTTS（AI API 语音）

当用户在 AI 配置面板中填入的 API 支持 TTS 时启用。原理：

```
浏览器 → fetch POST {base_url}/v1/audio/speech
         → 返回 MP3/WAV 音频流
         → AudioContext 或 <audio> 播放
```

标准请求体（兼容 OpenAI TTS）：

```json
{
  "model": "tts-1",
  "input": "要朗读的文本",
  "voice": "alloy",
  "response_format": "mp3"
}
```

用户可在配置面板中选择语音（`alloy` / `echo` / `fable` / `onyx` / `nova` / `shimmer` 等，视 API 而定）。

#### LocalModelTTS（本地模型）

用户本地跑了一个 TTS 推理服务（如 GPT-SoVITS、RVC、Coqui xTTS 等），暴露 HTTP 接口。

配置时需要填写：
- 服务地址（如 `http://localhost:9880`）
- 接口路径（如 `/tts`）
- 可选参数（说话人 ID、语速、情感等）

工具会先调接口获取可用语音/角色列表，再让用户选择。

#### TTS 选择优先级

```
用户打开设置
  ├── 系统检测到 AI API 支持 TTS → 列出 AiApiTTS 为候选项
  ├── 系统未检测到，但有本地服务配置 → 列出 LocalModelTTS
  └── 保底 → 始终可用的 WebSpeechTTS
```

用户在任何时候都可以手动切换。

**关键行为：** 当一个 `ScriptSection` 对应多张 slide 时，TTS 持续朗读不中断，只在翻页时保持同步。

### 模块 6：自动滚播控制器（状态机）

```
         ┌──────────┐
         │   IDLE    │  ← 初始状态
         └─────┬────┘
               │ 点击「播放」
               ▼
         ┌──────────┐    pause()    ┌──────────┐
         │ PLAYING  │──────────────►│ PAUSED   │
         └─────┬────┘              └────┬─────┘
               │ TTS onEnd              │ resume()
               │ 且还有下一 pair         │
               ├────────────────────────┘
               │
               ▼
         ┌──────────┐
         │ FINISHED │  ← 所有 pair 播完
         └──────────┘
```

**每个 AlignedPair 的播放逻辑：**

```
1. 显示该 pair 的 slides（第一页或全部）
2. 如果 scriptText 不为空：
   → TTS 开始朗读 scriptText
   → 如果有多页 slide，用户手动翻页或定时翻页，TTS 不中断
   → TTS onEnd → 进入下一 pair
3. 如果 scriptText 为空（空白插入）：
   → 停留 holdSeconds 秒
   → 自动进入下一 pair
```

### 模块 7：录制导出（选做，MVP 不包含）

同原方案，不做改动。

## 五、技术选型总结

| 层面 | MVP 选择 | 理由 |
|------|----------|------|
| 框架 | **Vue 3** (Composition API) | 生态成熟，文档齐全 |
| 构建 | **Vite** | 秒级 HMR，纯静态输出 |
| MD 解析 | **marked** | 轻量、快速、支持 GFM |
| 拖拽交互 | 原生 DragEvent +  Pointer Events | 对齐编辑器核心交互，避免引入重型库 |
| 代码高亮 | **highlight.js** (按需加载) | 一行集成 |
| TTS | **Web Speech API** | 浏览器内置，零依赖 |
| 录制 | 暂无（MVP 不做） | 降低初始复杂度 |
| 部署 | **GitHub Pages / Vercel** | 纯静态，无后端 |

## 七、用户完整工作流（分步演示）

### Step 1 — 导入脚本
```
[粘贴脚本] 或 [拖入 .txt 文件]
→ 文本显示在编辑区，按段落间距带默认分节线
→ 每节左侧显示「第1节」「第2节」…… 和文本预览
```

### Step 2 — 调整脚本分节
```
用户点击段落间的「分割/合并」按钮调整分节
→ 节编号自动更新
→ 用户确认分节无误
```

### Step 3 — 导入 Markdown
```
[粘贴 Markdown] 或 [拖入 .md 文件]
→ 系统解析出 Markdown 节列表，显示在右侧列
→ 左侧脚本列、右侧 Markdown 列并排呈现
```

### Step 4 — 对齐编辑
```
系统尝试自动匹配（按文本相似度打分）
→ 用户检查匹配结果
→ 拖拽修正不对的匹配
→ 在需要处插入空白（左侧或右侧）
→ 将过长的 Markdown 节拆为多页
→ 所有红色高亮消除 → 对齐完成
```

### Step 5 — 预览播放
```
点击「预览播放」
→ 进入全屏播放模式
→ 第一页幻灯片显示 + TTS 朗读对应脚本
→ 读完自动翻到下一对齐对

```
